API

The tracker module contains the Tracker class which can alternatively be imported directly from the gpu_tracker package.

class gpu_tracker.tracker.Tracker(sleep_time: float = 1.0, include_children: bool = True, ram_unit: str = 'gigabytes', gpu_ram_unit: str = 'gigabytes', time_unit: str = 'hours', n_join_attempts: int = 5, join_timeout: float = 10.0, kill_if_join_fails: bool = False, process_id: int | None = None)[source]

Runs a thread in the background that tracks the compute time, maximum RAM, and maximum GPU RAM usage within a context manager or explicit start() and stop() methods. Calculated quantities are scaled depending on the unit chosen for them (e.g. megabytes vs. gigabytes, hours vs. days, etc.).

Variables:

  • max_ram (float) – The highest RAM observed while tracking.

  • max_gpu_ram (float) – The highest GPU RAM observed while tracking.

  • compute_time (float) – The amount of time spent tracking.

Parameters:

  • sleep_time (float) – The number of seconds to sleep in between usage-collection iterations.

  • include_children (bool) – Whether to add the usage (RAM and GPU RAM) of child processes. Otherwise, only collects usage of the main process.

  • ram_unit (str) – One of ‘bytes’, ‘kilobytes’, ‘megabytes’, ‘gigabytes’, or ‘terabytes’.

  • gpu_ram_unit (str) – One of ‘bytes’, ‘kilobytes’, ‘megabytes’, ‘gigabytes’, or ‘terabytes’.

  • time_unit (str) – One of ‘seconds’, ‘minutes’, ‘hours’, or ‘days’.

  • n_join_attempts (int) – The number of times the tracker attempts to join its underlying thread.

  • join_timeout (float) – The amount of time the tracker waits for its underlying thread to join.

  • kill_if_join_fails (bool) – If true, kill the process if the underlying thread fails to join.

  • process_id (int | None) – The ID of the process to track (along with its children if include_children is set). Defaults to the current process.

Raises:

ValueError – Raised if invalid units are provided.
